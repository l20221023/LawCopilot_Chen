# LawCopilot 需求拆解

## 1. 项目目标

LawCopilot 是一个面向执业律师的在线 AI 实务助手。第一阶段目标不是做完整 SaaS，而是做一个可以稳定支撑小规模测试的可用版本，覆盖登录、场景化对话、附件上传、额度控制、模拟付费和基本设置。

当前用户规模约 10 人。用户不具备技术背景，因此系统必须满足以下约束：

- 无 API Key 配置入口
- 默认即可使用
- 功能入口清晰
- 关键状态可见，例如额度、订阅状态、当前场景
- 支持桌面端优先，移动端为加分项

## 2. 范围界定

### 2.1 本期必须实现

- Supabase Auth 邮箱密码注册登录
- 用户资料读取与展示
- 12 个法律实务场景
- 会话列表与消息持久化
- 文本输入、图片上传、PDF/TXT 上传
- AI 流式响应
- 额度校验与扣减
- 付费方案展示页
- 模拟购买流程
- 设置页
- token 输入输出记录

### 2.2 本期明确不做

- 真实支付接入
- 管理后台
- 多租户和团队协作
- 审计日志与细粒度权限
- 复杂知识库检索
- 法院/法规外部数据源实时接入

## 3. 需求按能力域拆解

### 3.1 认证与用户域

目标：让律师完成注册登录，并在系统内具备可追踪的订阅与额度状态。

包含：

- 注册
- 登录
- 登出
- session 持久化
- 用户扩展资料读取
- 订阅状态展示

依赖：

- Supabase 项目配置
- `users` 表扩展字段

### 3.2 对话域

目标：让用户基于某个法律场景完成一次完整 AI 对话，并可查看历史会话。

包含：

- 新建对话
- 切换对话
- 删除对话
- 消息流渲染
- Markdown 展示
- 重新生成
- 停止生成
- 消息持久化

依赖：

- 登录态
- 场景数据
- AI 请求层

### 3.3 场景域

目标：通过 12 个预设场景把通用模型包装成可直接使用的法律助手。

包含：

- 场景列表
- 场景说明
- 场景图标
- system prompt
- 默认场景设置
- 场景切换确认逻辑

依赖：

- 会话创建逻辑

### 3.4 附件域

目标：支持律师上传案件材料作为上下文输入。

包含：

- 图片上传与缩略图预览
- PDF 上传与文本提取
- TXT 上传与文本读取
- 附件卡片展示
- 附件数据写入消息

依赖：

- 前端文件读取
- Prompt 组装逻辑

### 3.5 额度与付费域

目标：在每次发送消息前完成业务拦截，保证免费、按次、订阅三类用户的计费规则一致执行。

包含：

- 发送前额度校验
- 发送后扣减
- 订阅有效期判断
- 额度不足弹窗
- 定价页
- 模拟购买流程

依赖：

- 用户状态
- 数据库更新

### 3.6 统计与记录域

目标：记录每次调用的输入输出 token，便于后续成本统计与产品优化。

包含：

- usage 数据结构
- 请求完成后记录 token
- 会话级或消息级关联

依赖：

- AI 返回 token 信息

## 4. 页面拆解

建议首期路由如下：

- `/auth`：登录/注册
- `/app/chat`：主对话页
- `/app/pricing`：定价页
- `/app/settings`：设置页

可以增加默认跳转：

- 未登录进入 `/auth`
- 已登录访问根路径跳到 `/app/chat`

## 5. 关键业务规则

### 5.1 免费用户

- 初始 5 次额度
- 每次发送前必须校验 `remaining_credits > 0`
- 发送成功后扣减 1
- 为 0 时必须拦截

### 5.2 按次用户

- 购买后增加 `remaining_credits`
- 发送成功后扣减 1
- 用完后按免费用户逻辑拦截

### 5.3 订阅用户

- 发送前判断 `subscription_expires_at > now`
- 未过期则不限次数
- 已过期则回退为 `free`

### 5.4 扣费时机

建议采用“发送成功后扣减”：

- 发送前做资格校验
- 生成完成或已经拿到有效响应后扣减
- 如果请求在发出前失败，不扣减

这样对用户更公平，也更容易解释。

## 6. 技术约束

- 前端：React + Vite + Tailwind CSS + React Router
- 后端能力：Supabase Auth + Database + Storage
- AI 接口：OpenRouter，兼容 OpenAI Chat Completions
- API Key 只能在服务端或 Supabase Edge Function 中使用
